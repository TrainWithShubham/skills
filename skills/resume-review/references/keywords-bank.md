# DevOps Keywords Bank

Use this as a reference vocabulary when analyzing a DevOps/SRE/Cloud/Platform resume. These are the terms that appear in most DevOps JDs and that ATS systems look for.

**When building the "Missing Keywords" section of a review:**
1. First extract keywords from the provided JD.
2. Cross-reference with these categories to spot obvious omissions the user might fill in.
3. Only suggest keywords the candidate plausibly has experience with (don't suggest they add "Kubernetes" if they've clearly never touched it — that's dishonest).

## How to recommend missing keywords

When presenting "Missing Keywords" in a review:

1. **Prefer JD keywords first.** If the JD says "Terraform" and the resume doesn't mention it, that's the top miss.
2. **Then suggest adjacent keywords from this bank** — but only if the candidate's experience plausibly covers them.
3. **Include both full name and abbreviation** where applicable: "Amazon Web Services (AWS)", "Kubernetes (K8s)", "Infrastructure as Code (IaC)".
4. **Don't suggest keyword stuffing.** If the candidate has no monitoring experience, don't recommend adding "Prometheus" to the skills list just to pass ATS. That will fail in the interview.

---

## Guidance for non-DevOps resumes

This bank is DevOps-focused. For a frontend, backend, data, or ML resume, use the keywords from the JD as the primary source and be honest in the review that your keyword coverage is based on the JD alone (not an extensive domain bank). Flag this limitation to the user in the "JD Match Analysis" section.
